The Problem

AI agents are in production.
Nothing is testing them.

A team ships a prompt tweak on Friday afternoon. By Monday morning, the agent is looping on a tool call, burning $800/hour in API costs and timing out every user request.
An update that fixes one edge case silently breaks three others. There are no tests for prompts. No one knew until a customer complained.
The agent hallucinates a destructive database command on a real user's live environment. The sandbox they were testing in never caught it because the sandbox was just a local notebook.
A perfectly working agent stops working entirely after the LLM provider silently updates the model. No regression suite. No alert. Just angry users.

"We merged the prompt change. It worked in our head. It didn't work in production."

What We're Building

A gym for AI agents — where they train before they compete.

SimGym is a simulation engine, prompt CI/CD pipeline, and trace ingestion flywheel that ensures no broken agent ever merges to production.

01
The Gym
A sandboxed simulation environment that replays your real production traces against any prompt change — before you ship it. Isolated. Deterministic. Honest.
02
Prompt CI/CD
GitHub-native integration that runs your agent's full benchmark suite on every PR. Blocks merges on regressions. Posts the diff — exactly what broke and why.
03
Shadow Loop
Ingests live production traces, flags anomalies overnight, generates a prompt fix, and opens the PR itself. You wake up to a solution, not an incident.
The North Star Experience
"SimGym found an edge case in your agent at 2 AM. It already wrote the prompt fix. It already verified the fix passes. Here's the PR — you just need to approve it."
